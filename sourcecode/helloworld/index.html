The Hello World Driver

This driver just prints debug messages. Don't forget to run the better DbgView (from sysinternals.com)!

The files:
  1. hello.c
  2. makefile
  3. sources

What does it do

This example has NO functionality other than to be loaded and unloaded. When it is loaded and unloaded, it deposits a print statement into the kernel's debug print buffer. This example is best run by using the Win32 API or the utility mentioned below. Also run DbgView so that you can see the print statements in that Kernel buffer.

How to build

After installing the DDK, open the build environment "Windows XP Free Build." Change to the directory with these files, then execute "build.exe." The build utility should be automatically in your path.

How to Install

  1. Create a usermode program that uses the API calls CreateService and StartService (see MSDN library).
  2. Use OSR's Driver Loader Utility instead of writing your own

What is each file

  1. makefile only points to the DDK's default build script
  2. sources contains a list of files to be included during compilation
  3. hello.c are the headers and code for the driver
